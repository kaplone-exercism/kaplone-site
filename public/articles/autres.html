Langages : autres

Je développe en java et en python mais j'essaye, autant que possible, de rester à d'autres langages.
Certains s'exécute sur la JVM, d'autres n'ont rien à voir avec java.

  • Scala :

    • Scala s'exécute sur la JVM et peut s'appuyer sur les bibliothèques java.
    • Scala apporte aussi ses propres librairies, certaines très puissantes.
    • Scala permettait un développement fonctionnel avant que java 8 ne fournisse une solution partielle.

  • Clojure :

    • Clojure s'exécute sur la JVM et peut s'appuyer sur les bibliothèques java.
    • Clojure est un langage concis, avec peu de règles de syntaxe. Sa syntaxe et ses principes sont proches de ceux de Lisp.

  • Racket :

    • Racket est un langage fonctionnel hérité de Scheme.
    • Comme Clojure, Racket a une syntaxe et des principes proches de ceux de Lisp.
    • DrRacket est un REPL/IDE qui facilite l'apprentissage du langage.

  • Go :

    • Go est un langage que j'ai découvert en cherchant une solution pour installer un dépôt git personnel sur un serveur : Gogs est écrit en Go.
    • J'ai aussi rencontré Go dans les outils pour exercism.io et syncthing (et docker ?).

  • TypeScript :

    • Typescript est un superSet de javascript lui apporte un typage statique fort.
