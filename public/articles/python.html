Langages : Python

Autodidacte depuis 2004 (j'ai du apprendre le langage python sur le tas pour améliorer le logiciel "ratatouille", initialement écrit par la ménagerie et diffusé dans leur "boîte à outils".
J'y ai appris à lire un code source, à comprendre comment la façon de penser d'un développeur influence et structure sa manière d'écrire une application.
Plus prosaïquement, j'ai découvert le langage python, la littérature informatique, la bibliothèque graphique GTK+, ainsi que FFmpeg et l'existence de linux

A partir de 2006, année de ma prise de poste chez satellite Multimédia j'ai été de plus en plus souvent en situation de devoir développer des scripts ou des petites applications.
Je commençais à me sentir familier avec python et j'ai continué avec python / GTK+ / ffmpeg. Je n'ai découvert que plus tard, en 2012, en commençant les cours au CNAM a quel point cette impression de familiarité pouvait être une illusion.

Cependant, Python, langage généraliste et bien documenté était un choix parfaitement raisonnable.
De plus, à cette époque, le typage dynamique ne semblait apporter que des avantages : moins de code à écrire, de décision à prendre, pas de dépassement de taille pour les entiers ...
Les subProcess demandaient un peu de travail quant à leurs contrôles, mais globalement je coulais des jours de développeur heureux.

En découvrant Java en 2012, Python est passé un peu au second plan. Convaincu par la richesse de l'écosystème autour de la JVM et surtout par certains avantages du typage statique, j'ai mis un peu Python de coté, ne conservant qu'un peu de maintenance et l'écriture de scripts coté serveur (Python est souvent installé par défaut sur les distributions linux).

Cependant, Python 3.6 introduit une notion de typage "par annotation" ou "par commentaire".
A partir de cette version, certains IDE proposent une vérification entre les types attendus et ceux constatés et signalent les incohérences au développeur.

Python présente encore de nombreux avantages :
Il est toujours possible de l'interpréter en mode interactif dans une console REPL.
Python propose de développer facilement dans des environnements virtuels (venv) qui peuvent encapsuler pour chaque projet des versions différentes des bibliothèques utilisées.

Aujourd'hui, la plus grande partie des défis que je code sur exercism.io est réalisé en Python.
