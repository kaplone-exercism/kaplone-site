Outils : Git

Git est un système de gestion de version.

Comment est-il possible aujourd'hui de travailler sans un système de gestion de version ?

Que l'on soit seul ou plusieurs sur un projet, que celui-ci soit public ou privé, que l'on préfère SVN ou GIT, peu importe, maitriser ses versions est essentiel.

Après avoir travaillé sans (l'inconscience de la jeunesse) j'ai d'abord commencé avec SVN avant de me tourner vers GIT.
En plus des dépots sur mon ordinateur local, j'ai également un compte github pour les source publiques et un serveur hébergeant Gogs qui fait un backup de github et mes dépots privés.

    Concrètement, Git permet :
  • Conserver une trace des évolutions du code et pouvoir naviguer entre elles.
  • Partager aisément les cources d'un projet en permettant la consultation, l'acquisition, le clonage, le forkage, ...
  • D'avoir une solution globale pour le suivi des bugs, pour les "pull requests".
  • D'organiser efficacement un travail collaboratif.
  • D'intégrer une chaîne d'intégration continue.
