Divers : JSON

Json est une forme de fichier (ou de flux) structuré pour permettre le stockage ou l'échange de données.

Globalement, Json est considéré comme plus facile à lire par les humains que son équivalent en XML.
L'expression de l'arbre à construite se résume à une succession/imbrications de tableaux associatifs et de listes.

Sa syntaxe BNF peut être résumée par les définitions suivantes :

  • begin-array = [ (left square bracket)
  • begin-object = { (left curly bracket)
  • end-array = ] (right square bracket)
  • end-object = } (right curly bracket)
  • name-separator = : (colon)
  • value-separator = , (comma)

MongoDb, la base de données orientée document stocke ses enregistrements en Bson. Un document retourné lors d'une requête se présente sous la forme d'un fichier Json.

Tous les langages offrent des solutions performantes pour construire / parser / mapper des données Json à partir ou vers une structure interne au langage.
En Java, j'utilise Jackson.
En Python il y a une correspondance assez nette entre les dictionnaires et les objets Json, et entre les listes et les arrays Json.
